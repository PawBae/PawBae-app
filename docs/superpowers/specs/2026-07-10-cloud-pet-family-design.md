# PawBae 云端萌宠家族 — 角色设计规格

**日期：** 2026-07-10
**阶段：** 首轮 3D 角色概念
**方向：** 天气性格组
**角色数量：** 4

## 目标

为 PawBae 建立一组与 Yoonie 同属云端世界的原创幻想萌宠。它们以性格建立情感连接，以天气气质塑造外形；不承担固定的 Agent 功能映射。首轮输出用于验证角色魅力、家族统一性和剪影辨识度，不直接作为桌面精灵生产资产。

## 创意原则

- 保留参考图中柔软绒毛、糖果配色、精致 3D 公仔和亲和表情带来的吸引力。
- 不复制参考图中的具体动物、服装、五官比例、角色轮廓或命名。
- 全员是原创幻想生物，只借用少量自然界特征，不明确归属于猫、兔、犬、鼠等现实物种。
- 性格优先于功能。Agent 状态未来通过动作和表情表达，不把角色变成状态图标。
- 每只角色须有独立的耳朵、尾巴和整体剪影，在缩小显示时仍能区分。

## 世界观关系

四只新角色与 Yoonie 来自 agent 们工作的同一片云。不同区域的天气会慢慢凝成不同气质的小生物。它们与 Yoonie 是朋友和邻居，不取代 Yoonie 的核心 IP 地位。

家族共同标记为“心形云核”：每只角色身上都隐藏一个简洁心形结构，可出现在胸口绒毛、额头纹样或尾巴末端。该标记呼应 PawBae 图标中的心形视觉资产。

## 角色阵容

### Solu / 小煦

- **物种称呼：** 晴绒兽
- **核心性格：** 乐观、外向、会主动照顾气氛
- **性格钩子：** 坚信坏事晒一晒就会变好
- **主色：** 奶油黄、蜜桃橙、少量珊瑚粉
- **轮廓：** 太阳花状耳冠，圆润面颊，短小四肢，蓬松逗号尾
- **云核位置：** 胸前浅色绒毛形成柔和心形
- **代表动作：** 身体前倾、尾巴高高卷起，像刚发现一件值得分享的好事
- **避免：** 不做成狮子、狐狸或向日葵拟人；不增加服装和道具

### Muru / 雾露

- **物种称呼：** 雾团兽
- **核心性格：** 安静、害羞、观察力强
- **性格钩子：** 喜欢躲在 Yoonie 的耳朵后面观察大家
- **主色：** 雾蓝、淡紫、珍珠白
- **轮廓：** 垂落的半透明云耳，身体像一团低矮软雾，尾巴边缘逐渐消散
- **云核位置：** 额头中央的雾白心形斑纹
- **代表动作：** 耳朵半遮住脸，只露出好奇的眼睛
- **避免：** 不做成兔子或幽灵；半透明只用于耳尖与尾端，主体保持温暖可触摸感

### Riffi / 雷栗

- **物种称呼：** 雷芽兽
- **核心性格：** 莽撞、热情、胆子其实很小
- **性格钩子：** 出场声音很大，受惊后会立刻缩成毛球
- **主色：** 薄荷绿、电光黄、少量深青蓝
- **轮廓：** 闪电形耳尖，圆豆身体，短圆四肢，锯齿状短尾
- **云核位置：** 尾巴末端的负形心形缺口
- **代表动作：** 一只脚刚迈出去，耳尖冒出细小静电绒光，表情兴奋又有点紧张
- **避免：** 不做成龙、皮卡丘式电系动物或带角怪兽；闪电语言保持柔软圆角

### Luma / 星沫

- **物种称呼：** 星露兽
- **核心性格：** 慵懒、爱幻想、浪漫大方
- **性格钩子：** 经常把梦里捡到的东西当作礼物送人
- **主色：** 莓粉、夜空蓝、少量柔金色
- **轮廓：** 月牙形侧耳，微微后仰的软身体，彗星状长尾，毛尖带稀疏星尘感
- **云核位置：** 尾巴靠近身体处的柔金色心形斑纹
- **代表动作：** 轻轻打哈欠，尾巴托着一颗像露珠的小光点
- **避免：** 不做成独角兽、星空猫或传统月兔；星光效果克制，不覆盖主体轮廓

## 统一视觉语言

- **比例：** 大头、短肢、低重心，整体接近可抱住的软质公仔；眼睛灵动但不过度巨大。
- **造型：** 以一到两个大形状建立剪影，减少细碎装饰；每只角色只保留一个主要视觉奇点。
- **材质：** 柔软短绒为主，耳尖、尾端或云气结构允许轻微半透明；避免塑料感和湿亮皮肤。
- **渲染：** 高完成度 3D 角色概念，柔和棚拍光，细腻但不过度写实的绒毛，轻微环境遮蔽。
- **表情：** 温和、真诚、有一点笨拙；避免媚笑、夸张潮酷姿势和成人化神态。
- **配色：** 糖果色但降低荧光饱和度，确保四只并排时既和谐又可独立识别。
- **品牌连接：** 心形云核是唯一强制共通符号，不统一服装，不添加品牌文字。

## 首轮概念板

生成一张横向 4 角色概念板，依次展示 Solu、Muru、Riffi、Luma。每只角色包含：

1. 完整正面或轻微三分之四视角全身像；
2. 一个能体现性格的自然动作；
3. 简洁英文名与中文名；
4. 独立的柔和纯色或轻渐变背景块。

画面整体保持高级玩具设计提案感，不放入手机界面、数值卡片、按钮或复杂场景。文字只出现角色名字，避免生成冗长说明。

## 验收标准

- 四只都能在不依赖文字和颜色的情况下通过剪影区分。
- 角色看起来属于同一世界，但不存在换色复用同一身体的感觉。
- 每只角色的第一印象与对应性格一致。
- 心形云核自然融入身体结构，不像后来贴上的徽章。
- 画面体现参考图的柔软、精致与糖果色亲和力，同时没有复制其具体角色设计。
- 概念板足以支持下一轮选择：保留、调整或淘汰任一角色。

## 首轮不做

- 不生成 spritesheet、逐帧动作或透明背景生产素材。
- 不设计服装、职业、数值、稀有度或领养 UI。
- 不为角色绑定固定 Agent 状态或工作职责。
- 不在首轮扩展超过四只角色，也不重做 Yoonie。
