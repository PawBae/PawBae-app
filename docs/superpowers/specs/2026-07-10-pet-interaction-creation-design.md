- **作者**:Yining + Claude(brainstorming 会话产出)

## 0. 范围与定位

本文档回答两个 v1.1/v2 方向问题:

1. **深度互动**——宠物如何与用户互动(对话、触摸、养成、工作联动)
2. **用户创造宠物**——用户如何获得独一无二的自己的宠物

**不占 v1 排期**,唯一例外:§7 官方四宠资产是 v1 发布刚需(替换版权占位皮肤),但它是纯资产工作,不动引擎代码。

## 1. 已定决策

| 决策点 | 结论 | 核心理由 |
|---|---|---|
| 落地时机 | v1.1/v2;v1 只做官方四宠资产 | v1 十周计划已锁死,串门是底线 |
| 互动核心 | AI 对话/陪伴为核心;物理触摸、养成深化、工作联动全要 | 呼应"让 AI agent 可亲近"愿景 |
| 创造形态 | AI 生成独一无二外观 | 差异化最强 |
| 动画标准 | 轻动效(静态立绘+程序化动效),不追逐帧 | 成本/成功率分水岭 |
| AI 大脑 | 复用用户自己的 coding agent(CLI 桥) | 零推理成本;"你的 agent 就是你的宠物"叙事 |
| 运行时渲染 | 统一 2D;不做实时 3D | WebView 实时 3D 画质倒退、UGC 管线相性差、常驻 GPU 成本 |
| 成本结构 | 对话=持续成本→用户 agent(免费);生成=一次性成本→云端(付费点"领养费") | 成本模型自洽 |

## 2. 总体架构:三轴模型与渲染器接口

产品表现力的三根轴:

- **身体**(渲染)——够可爱即可,不追世界第一;所有虚拟宠物公司共享的赛道
- **灵魂**(对话/性格/记忆)——卡位 coding agent 入口,天花板独有
- **手**(宠物替你做事,本身是 agent)——同上,远期轴

**核心架构不变式:「宠物状态机 → 视觉输出」接口保持稳定,渲染器可整体替换。**

```
状态机 (pet.svelte.ts / codex-pet 状态) ──┐
互动事件 (点击/拖拽/喂食)                 ├─→ [渲染器接口] ─→ SpritePet   (现有,图集逐帧)
情绪引擎 (灵魂层输出的情绪标签)           │                 ├─→ CapsulePet  (新增,立绘+程序动效)
pet-physics (位置/速度)                   ┘                 └─→ 未来 L3/L4 渲染器
```

状态机、互动、串门投影(GuestPet)、记忆全部渲染器无关。串门场景下,好友看到的 UGC 宠物由同一渲染器家族按胶囊清单渲染。

## 3. 宠物胶囊格式(Pet Capsule)

现有精灵包(`pet.json` + `spritesheet.webp`,见 public/assets/builtin/)的超集。胶囊 = 外观层 + 灵魂层 + 元数据:

```jsonc
{
  "id": "capsule-uuid",
  "schemaVersion": 3,
  "kind": "capsule",              // 区别于现有 "creature"
  "displayName": "泡泡",
  "animLevel": 0,                  // 动画等级,见下表
  "appearance": {
    "expressions": {               // L0:表情集(4~6 张静态透明 webp)
      "neutral": "neutral.webp",
      "happy": "happy.webp",
      "sleepy": "sleepy.webp",
      "sad": "sad.webp",
      "excited": "excited.webp"
    },
    "atlas": null,                 // L1+ 填充:生成的循环帧图集(复用现有 atlas 结构)
    "rig": null                    // L2+ 预留:分层绑骨数据
  },
  "soul": {
    "personaPrompt": "...",        // 领养时生成的性格卡
    "voice": { "style": "软糯", "quirks": ["句尾带波浪号"] },
    "traits": ["害羞", "嗜睡"]
  },
  "provenance": {                  // 确权与审核记录
    "generator": "pipeline-v1",
    "moderatedAt": "...",
    "sourcePromptHash": "..."
  }
}
```

**动画等级阶梯**(客户端按 `animLevel` 选择渲染路径;云端管线成熟一级解锁一级,老宠物可后台补渲升级——"你的宠物学会新动作了"可做运营事件):

| 等级 | 内容 | 状态 |
|---|---|---|
| L0 | 静态表情集 + 程序化动效 | v1.1 发布底线 |
| L1 | 生成循环帧(视频模型以立绘为首帧出 2~3s 呼吸/摇尾循环,抽帧成图集) | 增强,非依赖 |
| L2 | 自动分层绑骨(Live2D-lite) | 远期增强 |
| L3 | 实时风格化 3D(NPR 卡通渲染 + image→3D 管线) | 仅预留枚举值 |
| L4 | 生成式实时渲染(世界模型逐帧生成) | 仅预留枚举值 |

## 4. 渲染层:CapsulePet

新组件 `CapsulePet.svelte`,与 SpritePet 平级,**实现同一套 props 契约**(pet/state/size/loop/onOneShotEnd),因此 pet-physics、状态机、串门 GuestPet 零改动兼容。

**程序化动效清单**(CSS/WAAPI 变换,写一次全部胶囊宠物通用,零边际成本):

| 动效 | 触发 | 实现 |
|---|---|---|
| 呼吸 | 常驻 | 整图 scale 1.0↔1.02 缓动循环 |
| 待机漂浮/摇摆 | idle | translateY + 微 rotate 正弦 |
| 眨眼/表情切换 | 状态机 + 情绪标签 | 表情图交叉淡化(crossfade) |
| 拖拽挤压 | pointer 拖动 | squash & stretch,按拖拽速度形变 |
| 开心弹跳 | happy/excited | translateY 弹性曲线 |
| 移动滑行 | running 状态 | 位移 + 朝向倾斜 + 小幅 bounce |
| 触摸涟漪 | hover/click | 局部 scale 脉冲 + 心形粒子 |

**触摸反馈延迟目标 < 100ms**——情感表达性价比排序:反应速度 > 表情 > 肢体动画,这是本设计押注轻动效的依据。

## 5. 生成管线(云端,一次性,可收费)

```
用户输入(文字描述 / 参考照片)
  → 风格圣经模板拼 prompt(§7 沉淀)
  → 图像生成:主立绘(角色定稿,用户可重roll)
  → 图生图角色参考:生成 4~6 张表情变体
  → 毛发边缘 matting 抠图(BiRefNet 类)
  → 一致性 QC(自动:构图/色板校验;必要时人工抽检)
  → 内容审核(必过——UGC 会经串门投影给好友看)
  → 打包胶囊 → Supabase Storage → 客户端下载落地
```

**关键设计:漂浮系世界观约束。** UGC 宠物的生成模板天然约束为"漂浮系"物种(灵体/气泡精灵/云朵兽:无腿或短腿、圆润、悬浮姿态)。一石三鸟:

1. 无走路动画从"缺失的功能"变成"这个物种就这样"(与官方四宠有腿有帧的差异被世界观合法化)
2. 腿是图生图最易崩的部位,砍掉后表情集一致性显著提高
3. 造型圆润 → 程序化动效(呼吸/漂浮/挤压)表现力最大化

**失败与降级**:主立绘用户可重 roll(计费策略:N 次内免费);表情变体单张失败自动重试;整体失败率超阈值时降级到"固定骨架换肤"路线(预制漂浮系底模 + AI 生成花色纹理),保证交付。

**商业化**:生成是一次性成本 → 天然付费点(领养费/邀请奖励/次数包);对话是持续成本 → 用户自己的 agent 承担,免费。

## 6. 灵魂层:AI 对话与记忆

**大脑 = 用户自己的 coding agent。** 本地桥接 CLI(`claude -p` / `codex exec` 一类非交互调用),注入:

- 胶囊里的 `personaPrompt`(领养时由生成管线一并产出,与外观设定呼应)
- 本地记忆文件(宠物视角的事实库:主人叫什么、最近在做什么项目、上次聊了什么)
- 当日 agent 活动摘要(复用现有 hook 监控数据——**宠物看得见你的 agent 今天干了什么**,这是别人抄不了的差异化)

**输出双通道**:文本(升级现有 PetReplyBubble 为对话气泡)+ 情绪标签(`happy/sad/sleepy/excited/...`)驱动 §4 的表情切换——聊天时脸会变。

**记忆**:本地优先(JSONL/SQLite),不上云;串门场景只投影非隐私摘要(遵循串门设计的隐私投影原则)。对话内容永不进入云端。

**降级**:未检测到可用 agent 时,宠物退化为"哑巴但可爱"模式(动效+养成照常),对话入口显示接入引导。云端代理兜底留给面向大众的版本再决策,本设计不含。

## 7. 官方四宠(v1 刚需)

角色已定稿(2026-07-10,Yining 设计):

| 角色 | 中文名 | 元素 | 记忆点 | 性格种子(v1.1 persona 用) |
|---|---|---|---|---|
| Solu | 小煦 | 日 | 花瓣鬃毛 | 热情阳光 |
| Muru | 雾露 | 雾 | 云朵尾、垂耳 | 害羞治愈 |
| Riffi | 雷栗 | 雷 | 闪电耳/尾 | 元气冒失 |
| Luma | 星沫 | 星 | 星月角、星尘尾 | 嗜睡梦幻 |

**走现有 SpritePet 图集路线,零引擎改动**(现役 shimeji-bola 已证明低帧数成立:idle 1 帧、跑 4 帧)。美学定调:**羊毛毡定格动画**——2~4 帧切换是手工感而非缺陷;胖圆短腿体型用跳步(bounce-hop)走路。

**生产管线**(每只):主立绘 → 图像编辑模型小步幅出姿势/表情变体(每次只改一点)→ 3 倍候选人工策展 → BiRefNet 抠图 → 脚本对齐打包(统一画布、脚底基线对齐、webp 图集 + pet.json,cellW 384~512)→ 落 `public/assets/builtin/<id>/`。

帧数预算:idle 1~2、running 4、happy 2~3、eat 2、failed 2、working 2~3、其余状态 1~2 ≈ 20~25 张/只,四只共约 80~100 张策展图,≈1~2 周纯资产工作量。可选:每只用合格图训小 LoRA,后续扩展一致性大增(官方宠值得,UGC 不用)。

**⚠️ 版权占位下架(v1 发布阻断项)**:`public/assets/builtin/` 现存 naruto、nezuko、elaina、doro 等版权角色皮肤,闭测分发前必须移除,由官方四宠替代。

**战略红利**:① 此管线即 v1.1 UGC 管线的彩排;② 四宠元素设定直接成为 persona 种子;③ prompt 模板/色板/"每只一个记忆点"规则沉淀为**风格圣经**,即 §5 的生成模板。

## 8. 互动系统

在现有机制(HUNGER/AFFECTION/喂食/成长/番茄钟,pet.svelte.ts)上扩展,不推倒:

| 互动 | 输入 | 反馈 | 数值效果 |
|---|---|---|---|
| 抚摸 | hover 划过/长按 | 眯眼+心形粒子,<100ms | AFFECTION↑ |
| 拖拽 | pointer 拖动 | 挤压挣扎,松手弹回 | 心情短暂↓再恢复(never-punish:不扣好感) |
| 逗玩 | 点击 | 随机小反应池 | 心情↑ |
| 喂食 | 现有机制 | eat 表情/动画 | HUNGER 恢复 |
| 对话 | 气泡输入 | §6 双通道 | AFFECTION↑、记忆写入 |
| 工作联动 | agent 事件(现有 hook) | 状态表情:出错→委屈、完成→兴奋;可主动搭话("今天 agent 修了 3 个 bug 哦") | 情绪引擎输入 |

新增**情绪引擎**:纯函数状态机(仿 lease-machine.ts 的可测试风格),输入 = 互动事件 + agent 事件 + 养成数值,输出 = 情绪标签 → 渲染器表情。单元测试覆盖状态转移。

## 9. 天花板与演进判断

- **实时 3D**:不做(理由见 §1)。若做,唯一可行形态是 NPR 风格化渲染(绕开写实毛发),价值场景是手机 AR(v1.1 手机版之后再议)。
- **生成式实时渲染(L4)**:表现力终极形态,但距"三人公司可用"还有 2~4 年,且进度依赖外部实验室。只留枚举值。
- **战略判断**:视觉轴是共享赛道,永远做不到世界第一;灵魂+手两轴因卡位 coding agent 入口而天花板独有。**资源永远优先灵魂轴,身体轴"不拖后腿"即可。**

## 10. 法律与 IP

- 官方四宠与 UGC 均为"AI 初稿 + 人工精修定稿",**保留创作过程记录**(prompt、迭代版本、人工修改稿)以支撑确权;纯 AI 输出的版权保护在中美均不确定,不赌。
- 四宠名称(Solu/Muru/Riffi/Luma 及中文名)建议在商业化前评估商标注册。
- UGC 必过内容审核(§5),因串门会投影给好友;审核记录写入胶囊 `provenance`。
- 参考外部设计只取风格方向(风格不受版权保护),不复刻具体造型特征组合。

## 11. 里程碑切分与开放问题

**v1(十周内,纯资产)**:官方四宠图集制作 + 版权占位下架。无引擎改动。

**v1.1(建议切法)**:
1. CapsulePet 渲染器 + 程序动效库(纯前端,可用官方宠立绘先行开发)
2. 情绪引擎 + 触摸互动(纯本地)
3. agent CLI 对话桥 + persona/记忆(本地)
4. 云端生成管线 + 审核 + 领养流程(依赖 A 线云端排期)

1→2→3 可在无云端依赖下顺序交付;4 独立成线。

**开放问题**(实施规划时拍板):
- 领养定价与重 roll 免费次数
- agent CLI 桥的具体调用形态(每消息冷启动 vs 常驻会话)与多 agent(Claude/Codex/Cursor)适配优先级
- 表情集的标准表情枚举(建议先锁 neutral/happy/sleepy/sad/excited 五个)
- L1 视频生成循环帧的供应商与质量门槛
- 手机端(v1.1)是否首发支持胶囊宠物
