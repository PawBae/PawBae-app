# PawBae v1 双账号手动验收

这份清单用于自动化测试全部通过后的最终真人验收。除安装、启动和查看发布页外，
验收过程不得使用 DevTools、SQL 或手工修改本地存储。

## 手测前门槛

- 两台电脑安装同一份已签名的 beta 构建，分别使用两个真实 GitHub 账号。
- 生产 Supabase 已通过 `cloud deploy` 工作流部署；GitHub OAuth 已允许
  `http://127.0.0.1:53682/pawbae-auth` 回调。
- A 账号尚未兑换邀请码且持有一个有效邀请码；B 账号已在服务端保留邀请码资格。
  两端都从全新本地安装开始，以同时覆盖首次兑换和资格恢复。
- 另准备一轮全新本地安装验证“暂不登录”；不得通过 DevTools 或手改存储伪造状态。
- 生产 `pawbae.ai`、候补名单和 `/update/latest.json` 可访问。
- 两台电脑均开启 PawBae 的联机状态分享；测试期间不要打开数据库或开发工具。

## 1. 登录、重启与领养

1. A 点击 GitHub 登录，完成浏览器授权；核对 App 显示的 handle、头像和账号一致，
   并确认自动进入邀请码页。
2. A 从邀请码页返回 GitHub 页，再点击“暂不登录”；确认进入 Agent 页。随后点击返回，
   应回到 GitHub 页，且刚才的账号资料已清除、登录按钮重新出现。
3. 在单独一轮全新本地安装中完成上一步旁路流程，继续完成 Solu 领养；打开设置，
   确认账号仍为未登录。该轮只验收本地模式，不用于后续双账号串门。
4. A 在正式验收轮重新登录，进入邀请码页，兑换有效邀请码并进入 Agent 页。
5. B 使用已经兑换过邀请码的 GitHub 账号登录；确认自动跳过邀请码页并进入 Agent 页，
   不要求重复输入邀请码。
6. 分别从 Agent 页点击返回：A 应回到邀请码页，B 应回到 GitHub 页；两端都应能继续
   完成流程，不需要重新授权或重复兑换。
7. A、B 各自完成 Solu 领养；确认未完成的 Muru、Riffi、Luma 不可选。
8. 完全退出并重启 App。

通过标准：

- 登录过程有明确的加载、失败和重试状态，不会假装成功。
- 未兑换账号进入邀请码页；已有资格的账号跳过邀请码页。
- “暂不登录”会清除本次登录状态，返回路径与用户实际经过的页面一致。
- 从任一返回页面都能继续完成 onboarding，不会卡在只有账号资料、没有前进操作的页面。
- 重启后仍保持登录和邀请码已兑换状态，不重复要求邀请码。
- Onboarding、Home 和桌面上的宠物都是同一只 Solu，名称和皮肤一致。

## 2. 好友关系

1. A 用 B 的完整 GitHub handle 搜索；再用一段前缀搜索一次。
2. A 发送好友请求，B 在待处理列表接受。
3. B 对 A 执行静音、取消静音；确认关系仍在。

通过标准：

- 完整 handle 精确命中；前缀或不存在的 handle 不返回其他用户。
- A 显示“已发送”，B 显示“待接受”，接受后双方均显示好友。
- 静音不会解除好友，也不会暴露邮箱、仓库、任务文本或文件路径。

## 3. 串门主流程

1. A 向 B 发起串门，首次出现投影同意时确认。
2. B 先拒绝一次，确认双方回到可再次发起的稳定状态。
3. A 再次发起，B 接受。
4. A 运行一次真实 Agent 任务，并让其依次出现工作、等待确认、完成/空闲状态。

通过标准：

- 请求、拒绝、再次请求、接受都只需使用 App UI。
- 接受后 A 的桌面显示宠物离家/去向和召回入口，不出现第二只本地分身。
- B 的桌面同时出现自己的 Solu 与 A 的 Solu，身份标签清楚。
- 访客宠物能反映有限的工作、等待、空闲/离线状态，但没有任何工作内容。
- Home、A 桌面、B 桌面都使用相同 Solu 资源，没有空白帧、跳尺寸或错误皮肤。

## 4. 召回、断网与撤权

依次建立三次串门来验证：

1. A 在访问中点击召回。
2. 新访问中让 A 断网或退出 App，等待 B 看到访客离线；恢复网络并重启 A。
3. 新访问中由 B 拉黑 A；另建好友关系后再验证解除好友。

通过标准：

- 召回后双方最终都撤下访客，A 的 Solu 只回到自己的桌面。
- 断网时 B 只看到安全的离线状态；重连后不分身、不永久卡在访问中。
- 拉黑或解除好友会撤销访问和投影权限；被撤权后旧状态不继续更新。
- 任一步骤重复点击或重试都不会创建两张活动访问。

## 5. 正常结束与共同记忆

1. 再完成一次已开始的访问，用“结束访问”或等待 30 分钟自然结束。
2. A、B 分别打开相册/共同记忆。
3. 切换中英文各查看一次。

通过标准：

- 双方各得到且只得到一条同源记忆，重复刷新不会重复生成。
- 记忆只包含批准的模板、双方宠物身份和时间，不含任务文本、仓库或文件路径。
- 中英文表达像宠物回忆，不像内部状态或审计日志。

## 6. 官网、安装与更新

1. 在手机、平板和桌面浏览器打开 `pawbae.ai`。
2. 核对首屏真实 Solu、双宠物画面、closed beta 文案和下载入口。
3. 用已安装的旧 beta 检查更新，安装新 beta 后重新打开。

通过标准：

- 375、768、1440 宽度无横向溢出、按钮裁切或空白宠物图。
- 候补名单可提交，更新清单可访问。
- macOS Apple silicon / Intel 与 Windows 下载到正确架构的签名安装包。
- 更新器拒绝无签名或被修改的安装包，正常更新后版本号正确。
